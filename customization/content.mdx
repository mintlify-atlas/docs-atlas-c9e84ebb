## Overview

This guide shows you how to customize the content throughout your portfolio, including projects, personal information, social links, work experience, and skills.

### Adding a New Social Platform

1. Add the icon import at the top of `homepage.dart`:

```dart
import 'package:font_awesome_flutter/font_awesome_flutter.dart';
import 'package:simple_icons/simple_icons.dart';
```

2. Add to the social links array:

```dart

  'tooltip': 'YouTube',
  'icon': FontAwesomeIcons.youtube,
  'url': 'https://youtube.com/@yourchannel',
  'color': AppColors.youtube
},
```

## Projects

### Adding a New Project

Projects are defined in `lib/pages/projects.dart:48-110`. Add a new `ProjectCard` to the list:

```dart
final List<ProjectCard> projectCards = const [
  ProjectCard(
    projectName: "Your Project Name",
    projectDescription: "A brief description of what your project does and its key features.",
    imagePath: "assets/images/your-project.png",
    projectLogo: "assets/images/your-logo.png", // Optional
    projectLink: "https://github.com/yourusername/project",
    languages: ["Flutter", "Dart", "Firebase"],
    buttonText: "View on GitHub",
  ),
  // ... existing projects
];
```

### Project Card Properties

<Accordion title="ProjectCard Parameters">
- **projectName** - Display name of the project
- **projectDescription** - Short description (max 3 lines displayed)
- **imagePath** - Path to project screenshot in `assets/images/`
- **projectLogo** - Optional path to project logo/icon
- **projectLink** - URL to project (GitHub, live site, etc.)
- **languages** - Array of technology tags
- **buttonText** - CTA button text (e.g., "View on GitHub", "Open in Notion")
</Accordion>

### Technology Tags

The `languages` array creates colored tags. Common examples:

```dart
languages: [
  "Flutter",
  "Dart",
  "Firebase",
  "Figma",
  "REST APIs",
  "Postman",
  "Git",
  "CLI"
]
```

### Project Assets

1. Add your project image to `assets/images/`
2. Update `pubspec.yaml` if needed:

```yaml
flutter:
  assets:
    - assets/images/your-project.png
    - assets/images/your-logo.png
```

## Content Best Practices

<Accordion title="Writing Effective Project Descriptions">
1. **Be concise** - Keep descriptions under 3 lines
2. **Highlight impact** - What problem does it solve?
3. **Use action verbs** - Built, designed, implemented
4. **Include metrics** - "10+ articles", "4 speaking engagements"
</Accordion>

<Accordion title="Organizing Projects">
1. **Order by importance** - Put your best work first
2. **Variety** - Show different skills and technologies
3. **Recent work** - Keep portfolio up-to-date
4. **Quality over quantity** - 4-6 strong projects better than 10+ weak ones
</Accordion>

<Accordion title="Professional Links">
1. **Keep links active** - Test all URLs regularly
2. **Use UTM parameters** - Track portfolio traffic
3. **Professional consistency** - Use same username across platforms
4. **Privacy** - Only share what you're comfortable with
</Accordion>

## Testing Content Changes

After making content changes:

1. **Hot reload** during development: Press `r` in terminal
2. **Hot restart** for larger changes: Press `R` in terminal
3. **Full rebuild** for asset changes:

```bash
flutter run -d chrome
```
