The portfolio features a complete **dark/light theme system** powered by Flutter's Material 3 theming and GetX's theme management, with smooth transitions and theme-aware components.

## Theme Configuration

Both themes are defined in the main application setup:

```dart title="lib/main.dart:30-44"
GetMaterialApp(
  debugShowCheckedModeBanner: false,
  title: 'Himanshu Balani Portfolio',
  themeMode: ThemeMode.light,
  
  theme: ThemeData(
    brightness: Brightness.light,
    useMaterial3: true,
    fontFamily: GoogleFonts.outfit(fontWeight: FontWeight.bold).fontFamily,
  ),
  
  darkTheme: ThemeData(
    brightness: Brightness.dark,
    useMaterial3: true,
    fontFamily: GoogleFonts.outfit(fontWeight: FontWeight.bold).fontFamily,
  ),
  
  routes: {...},
  home: const HomePage(),
)
```

<Note>
  Both themes use the **Outfit** font family from Google Fonts with bold weight for a strong, modern aesthetic that aligns with the neobrutalism design.
</Note>

## Theme Toggle Component

Users can switch themes with an animated toggle button:

```dart title="lib/widgets/dark_mode_toggle.dart"
class DarkModeToggle extends StatelessWidget {
  const DarkModeToggle({super.key});

  @override
  Widget build(BuildContext context) {
    bool isDark = Theme.of(context).brightness == Brightness.dark;

    return GestureDetector(
      onTap: () {
        Get.changeThemeMode(isDark ? ThemeMode.light : ThemeMode.dark);
      },
      child: RepaintBoundary(
        child: SizedBox(
          width: 56,
          height: 56,
          child: Center(
            child: Tooltip(
              message: isDark ? "Dawn":"Dusk",
              decoration: TooltipStyles.decoration(isDark: isDark),
              textStyle: TooltipStyles.textStyle(isDark: isDark),
              child: Transform.flip(
                flipY: true,
                child: AnimatedSwitcher(
                  duration: const Duration(milliseconds: 300),
                  transitionBuilder: (child, animation) {
                    return ScaleTransition(
                      scale: animation,
                      child: RotationTransition(
                        turns: animation,
                        child: child,
                      ),
                    );
                  },
                  child: Icon(
                    isDark ? Icons.wb_incandescent_outlined 
                           : Icons.lightbulb_outline_rounded,
                    key: ValueKey(isDark),
                    size: 28,
                    color: isDark ? AppColors.brightOrange : Colors.black,
                    semanticLabel: isDark ? 'Light Mode' : 'Dark Mode',
                  ),
                ),
              ),
            ),
          ),
        ),
      ),
    );
  }
}
```

## Best Practices

<CardGroup cols={2}>
  <Card title="Check Theme in Build" icon="circle-check">
    Always read theme inside `build()` method so widgets rebuild when theme changes.
    
    ```dart
    // ✅ Correct
    @override
    Widget build(BuildContext context) {
      final isDark = Theme.of(context).brightness == Brightness.dark;
      return Container(color: isDark ? Colors.black : Colors.white);
    }
    
    // ❌ Wrong
    class MyWidget extends StatefulWidget {
      @override
      void initState() {
        final isDark = Theme.of(context).brightness == Brightness.dark;
        // Won't update when theme changes!
      }
    }
    ```
  </Card>
  
  <Card title="Use AppColors" icon="palette">
    Reference `AppColors` constants instead of hardcoded values for maintainability.
    
    ```dart
    // ✅ Correct
    color: AppColors.primaryPurple
    
    // ❌ Wrong
    color: Color(0xFF90a8ed)
    ```
  </Card>
  
  <Card title="Theme Utilities" icon="wrench">
    Create utility classes like `TooltipStyles` for commonly themed UI patterns.
  </Card>
  
  <Card title="Semantic Naming" icon="tag">
    Use descriptive color names like `brightOrange` instead of generic names like `accent1`.
  </Card>
</CardGroup>

## Customizing Themes

To modify the theme colors:

1. **Update color palette** in `lib/consts/style.dart:129-194`
2. **Adjust theme configurations** in `lib/main.dart:32-44`
3. **Test both themes** to ensure contrast and readability
4. **Verify component adaptations** across app bar, tooltips, backgrounds

<Note>
  Material 3 provides automatic color adaptations for elevation and states. Test interactive elements (buttons, cards) in both themes to ensure they remain visually distinct.
</Note>
