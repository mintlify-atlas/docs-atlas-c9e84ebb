        final baseCx = (col + 1) * cellWidth;

        // Horizontally displace alternate rows

## Hover Effects

The `OnHover` widget provides smooth transform animations on mouse enter/exit:

```dart title="lib/presentation/onhover.dart"
class OnHover extends StatefulWidget {
  final Widget Function(bool isHovered) builder;
  const OnHover({super.key, required this.builder});
}

class OnHoverState extends State<OnHover> {
  bool isHovered = false;

  @override
  Widget build(BuildContext context) {
    final hovered = Matrix4.identity()..translate(0, -10.0, 0);
    final transform = isHovered ? hovered : Matrix4.identity();

    return MouseRegion(
      onEnter: (_) => onEntered(true),
      onExit: (_) => onEntered(false),
      child: AnimatedContainer(
        duration: const Duration(milliseconds: 200),
        transform: transform,
        child: widget.builder(isHovered),
      ),
    );
  }

  void onEntered(bool isHovered) {
    setState(() {
      this.isHovered = isHovered;
    });
  }
}
```

<Accordion title="How the sine wave animation works">
  1. User hovers for 500ms
  2. Text fades out (10% of animation)
  3. Wave fades in while animating (20% of animation)
  4. Wave continues animating at full opacity (60% of animation)
  5. Wave fades out (20% of animation)
  6. Text fades back in (10% of animation)
  
  The result is a smooth, professional effect where the text briefly disappears while a sine wave animates through it, then the text returns.
</Accordion>

## Pulsing Animation

A simple scale animation for drawing attention to elements:

```dart title="lib/presentation/pulsingwrapper.dart:16-35"
class _PulsingWrapperState extends State<_PulsingWrapper> 
    with SingleTickerProviderStateMixin {
  late AnimationController _controller;
  late Animation<double> _animation;

  @override
  void initState() {
    super.initState();
    _controller = AnimationController(
      duration: widget.duration,
      vsync: this,
    )..repeat(reverse: true);

    _animation = Tween<double>(
      begin: 1.0,
      end: widget.maxScale,
    ).animate(CurvedAnimation(
      parent: _controller,
      curve: Curves.easeInOut,
    ));
  }
}
```

## Animation Best Practices

<CardGroup cols={2}>
  <Card title="Use Tickers Wisely" icon="clock">
    Always mix in `SingleTickerProviderStateMixin` when creating custom animations with `AnimationController`.
  </Card>
  
  <Card title="Dispose Controllers" icon="trash">
    Always call `_controller.dispose()` in the `dispose()` method to prevent memory leaks.
    
    ```dart
    @override
    void dispose() {
      _controller.dispose();
      super.dispose();
    }
    ```
  </Card>
  
  <Card title="Optimize with RepaintBoundary" icon="paintbrush">
    Wrap frequently animating widgets in `RepaintBoundary` to isolate repaints.
    
    ```dart
    RepaintBoundary(
      child: AnimatedWidget(...),
    )
    ```
  </Card>
  
  <Card title="Use Curves" icon="bezier-curve">
    Apply easing curves for more natural motion:
    
    ```dart
    CurvedAnimation(
      parent: _controller,
      curve: Curves.easeInOutCubic,
    )
    ```
  </Card>
</CardGroup>

## Performance Considerations

<Note>
  The animated pills background uses a `CustomPainter` which is highly efficient for drawing many shapes. The `shouldRepaint` method returns `true` because the animation is continuous, but Flutter's rendering pipeline only repaints when the frame changes.
</Note>

### Optimization Techniques Used

1. **CustomPainter for backgrounds** - More efficient than individual widgets
2. **RepaintBoundary isolation** - Prevents unnecessary repaints of static content
3. **AnimatedBuilder** - Only rebuilds animated portions of the widget tree
4. **Hover delays** - Prevents animation spam from mouse movements
5. **ValueKey for AnimatedSwitcher** - Ensures smooth icon transitions in theme toggle
