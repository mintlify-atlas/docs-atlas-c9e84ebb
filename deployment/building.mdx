## Overview

Building your Flutter portfolio for production involves compiling your Dart code into optimized JavaScript that can be deployed to any web hosting platform. This guide covers the build process, optimization options, and output structure.

## Basic Build Command

The simplest way to build your Flutter web app for production:

```bash
flutter build web
```

This command generates optimized web assets in the `build/web` directory, ready for deployment.

## Build with Optimization Flags

<Steps>
  <Step title="Standard Production Build">
    Use the release mode for optimized, minified output:
    
    ```bash
    flutter build web --release
    ```
    
    This is the default mode and produces the smallest bundle size.
  </Step>

  <Step title="Custom Base Path">
    If deploying to a subdirectory (e.g., GitHub Pages with repo name):
    
    ```bash
    flutter build web --base-href "/repository-name/"
    ```
    
    <Warning>
      The base href must start and end with a forward slash `/`.
    </Warning>
    
    For this portfolio (himanshubalani.github.io):
    
    ```bash
    flutter build web --base-href "/"
    ```
  </Step>

  <Step title="Web Renderer Selection">
    Choose between CanvasKit and HTML renderers:
    
    <CodeGroup>
    ```bash CanvasKit (Recommended)
    flutter build web --web-renderer canvaskit
    ```
    
    ```bash HTML Renderer
    flutter build web --web-renderer html
    ```
    
    ```bash Auto (Default)
    flutter build web --web-renderer auto
    ```
    </CodeGroup>
    
    <Tip>
      **CanvasKit** provides consistent rendering across browsers but has a larger bundle size (~2MB). **HTML** is lighter but may have rendering inconsistencies. **Auto** uses HTML on mobile and CanvasKit on desktop.
    </Tip>
  </Step>
</Steps>

## Advanced Build Options

### Tree Shaking and Optimization

```bash
flutter build web --release --tree-shake-icons
```

This removes unused Material and Cupertino icons from your bundle, significantly reducing size.

### Source Maps for Debugging

```bash
flutter build web --source-maps
```

<Warning>
  Only use source maps for debugging. They expose your source code and increase bundle size.
</Warning>

### Complete Production Build

Combine multiple flags for the optimal production build:

```bash
flutter build web \
  --release \
  --web-renderer auto \
  --base-href "/" \
  --tree-shake-icons
```

## Build Performance Tips

<Tip>
  **Speed up builds**:
  - Use `flutter build web --release --dart-define=FLUTTER_WEB_USE_SKIA=false` for faster HTML renderer builds
  - Enable web compilation cache: `flutter config --enable-web`
  - Run `flutter pub get` before building to ensure dependencies are cached
</Tip>

## Troubleshooting

### Build Fails

1. Clean the build cache:
   ```bash
   flutter clean
   flutter pub get
   flutter build web
   ```

2. Check Flutter version:
   ```bash
   flutter --version
   flutter doctor -v
   ```

### Large Bundle Size

1. Enable tree shaking:
   ```bash
   flutter build web --tree-shake-icons
   ```

2. Remove unused dependencies from `pubspec.yaml`

3. Use `--web-renderer html` for smaller builds (trade-off: rendering consistency)

### Asset Loading Issues

Ensure all assets in `pubspec.yaml` exist:
```yaml
flutter:
  assets:
    - assets/images/
```
