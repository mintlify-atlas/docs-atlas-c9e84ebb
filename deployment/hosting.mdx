## Overview

Your Flutter portfolio can be deployed to any static hosting platform. This guide covers the most popular options with complete configuration examples.

## GitHub Pages

GitHub Pages offers free hosting with custom domain support, perfect for portfolio websites.

<Steps>
  <Step title="Build for GitHub Pages">
    Build your app with the correct base href:
    
    <CodeGroup>
    ```bash Custom Domain
    # If using custom domain (e.g., himanshubalani.com)
    flutter build web --release --base-href "/"
    ```
    
    ```bash Repository Domain
    # If using github.io/repository-name
    flutter build web --release --base-href "/repository-name/"
    ```
    </CodeGroup>
  </Step>

  <Step title="Add 404 Handling">
    GitHub Pages needs special 404 handling for Flutter's routing. Your portfolio already includes a custom 404.html from `web/404.html`.
    
    The 404 page features:
    - Neobrutalism design matching your theme
    - Background color: `#90A8ED`
    - Custom 4🐼4 error message
    - Redirect button to home
    
    <Tip>
      The custom 404 page is automatically included in your build output.
    </Tip>
  </Step>

  <Step title="Deploy to GitHub Pages">
    <CodeGroup>
    ```bash Manual Deployment
    # Navigate to build output
    cd build/web
    
    # Initialize git (if not already)
    git init
    git add .
    git commit -m "Deploy to GitHub Pages"
    
    # Push to gh-pages branch
    git branch -M gh-pages
    git remote add origin https://github.com/username/repository.git
    git push -u origin gh-pages --force
    ```
    
    ```yaml GitHub Actions
    # .github/workflows/deploy.yml
    name: Deploy to GitHub Pages
    
    on:
      push:
        branches: [ main ]
    
    jobs:
      deploy:
        runs-on: ubuntu-latest
        steps:
          - uses: actions/checkout@v3
          
          - name: Setup Flutter
            uses: subosito/flutter-action@v2
            with:
              flutter-version: '3.19.0'
              channel: 'stable'
          
          - name: Install dependencies
            run: flutter pub get
          
          - name: Build web
            run: flutter build web --release --base-href "/"
          
          - name: Deploy to GitHub Pages
            uses: peaceiris/actions-gh-pages@v3
            with:
              github_token: ${{ secrets.GITHUB_TOKEN }}
              publish_dir: ./build/web
    ```
    </CodeGroup>
  </Step>

  <Step title="Configure Custom Domain (Optional)">
    Add a `CNAME` file to `web/` directory:
    
    ```txt web/CNAME
    himanshubalani.com
    ```
    
    Then configure DNS:
    ```
    Type: A
    Name: @
    Value: 185.199.108.153
           185.199.109.153
           185.199.110.153
           185.199.111.153
    
    Type: CNAME
    Name: www
    Value: username.github.io
    ```
  </Step>
</Steps>

## Firebase Hosting

Firebase Hosting provides fast CDN, automatic SSL, and easy rollbacks.

<Steps>
  <Step title="Install Firebase CLI">
    ```bash
    npm install -g firebase-tools
    firebase login
    ```
  </Step>

  <Step title="Initialize Firebase">
    ```bash
    firebase init hosting
    ```
    
    When prompted:
    - Public directory: `build/web`
    - Configure as single-page app: `Yes`
    - Set up automatic builds: `No` (we'll build manually)
  </Step>

  <Step title="Configure firebase.json">
    Create or update `firebase.json`:
    
    ```json firebase.json
    {
      "hosting": {
        "public": "build/web",
        "ignore": [
          "firebase.json",
          "**/.*",
          "**/node_modules/**"
        ],
        "rewrites": [
          {
            "source": "**",
            "destination": "/index.html"
          }
        ],
        "headers": [
          {
            "source": "**/*.@(jpg|jpeg|gif|png|svg|webp|js|css|woff|woff2)",
            "headers": [
              {
                "key": "Cache-Control",
                "value": "max-age=31536000"
              }
            ]
          }
        ]
      }
    }
    ```
  </Step>

  <Step title="Build and Deploy">
    ```bash
    flutter build web --release
    firebase deploy --only hosting
    ```
    
    Your site will be live at: `https://project-id.web.app`
  </Step>
</Steps>

<Tip>
  Firebase Hosting includes automatic SSL certificates and global CDN distribution.
</Tip>

## Netlify

Netlify offers continuous deployment, instant rollbacks, and split testing.

<Steps>
  <Step title="Create netlify.toml">
    Add configuration to your repository root:
    
    ```toml netlify.toml
    [build]
      command = "flutter build web --release"
      publish = "build/web"
    
    [build.environment]
      FLUTTER_VERSION = "3.19.0"
    
    [[redirects]]
      from = "/*"
      to = "/index.html"
      status = 200
    
    [[headers]]
      for = "/*"
      [headers.values]
        X-Frame-Options = "DENY"
        X-XSS-Protection = "1; mode=block"
        X-Content-Type-Options = "nosniff"
        Referrer-Policy = "strict-origin-when-cross-origin"
    
    [[headers]]
      for = "/assets/*"
      [headers.values]
        Cache-Control = "public, max-age=31536000, immutable"
    ```
  </Step>

  <Step title="Deploy via Git">
    <CodeGroup>
    ```bash New Site
    # Push your code to GitHub/GitLab/Bitbucket
    git add netlify.toml
    git commit -m "Add Netlify configuration"
    git push
    
    # Connect repository on Netlify dashboard
    # Netlify will auto-detect Flutter and deploy
    ```
    
    ```bash Netlify CLI
    npm install -g netlify-cli
    netlify login
    
    # Build locally
    flutter build web --release
    
    # Deploy
    netlify deploy --prod --dir=build/web
    ```
    </CodeGroup>
  </Step>

  <Step title="Custom Domain">
    In Netlify dashboard:
    1. Go to **Domain settings**
    2. Add custom domain: `himanshubalani.com`
    3. Configure DNS:
       ```
       Type: CNAME
       Name: www
       Value: your-site.netlify.app
       ```
  </Step>
</Steps>

## Vercel

Vercel provides zero-config deployments with global edge network.

<Steps>
  <Step title="Install Vercel CLI">
    ```bash
    npm install -g vercel
    vercel login
    ```
  </Step>

  <Step title="Create vercel.json">
    ```json vercel.json
    {
      "buildCommand": "flutter build web --release",
      "outputDirectory": "build/web",
      "framework": null,
      "devCommand": null,
      "installCommand": "if [ ! -d flutter ]; then git clone https://github.com/flutter/flutter.git -b stable --depth 1; fi && flutter/bin/flutter doctor && flutter/bin/flutter pub get",
      "routes": [
        {
          "handle": "filesystem"
        },
        {
          "src": "/(.*)",
          "dest": "/index.html"
        }
      ],
      "headers": [
        {
          "source": "/assets/(.*)",
          "headers": [
            {
              "key": "Cache-Control",
              "value": "public, max-age=31536000, immutable"
            }
          ]
        }
      ]
    }
    ```
  </Step>

  <Step title="Deploy">
    ```bash
    # First deployment
    vercel
    
    # Production deployment
    vercel --prod
    ```
    
    Or connect your Git repository for automatic deployments.
  </Step>
</Steps>

<Warning>
  Vercel's free tier has bandwidth limits. Monitor usage for high-traffic portfolios.
</Warning>

## Performance Optimization

After deployment, optimize your portfolio:

### Enable Compression

Most platforms support automatic gzip/brotli compression. Verify with:

```bash
curl -H "Accept-Encoding: gzip,deflate" -I https://your-domain.com
```

Look for `Content-Encoding: gzip` header.

### Configure Caching

Optimal cache headers are already included in the hosting configurations above:
- Assets: 1 year cache (`max-age=31536000`)
- HTML: No cache (always fresh)

### CDN Distribution

All recommended platforms provide global CDN:
- **GitHub Pages**: Fastly CDN
- **Firebase**: Google's global network
- **Netlify**: Netlify Edge network
- **Vercel**: Vercel Edge Network

### Monitor Performance

Use these tools to track your portfolio:

```bash
# Lighthouse audit
npm install -g lighthouse
lighthouse https://your-domain.com --view

# Core Web Vitals
# Check at https://pagespeed.web.dev/
```

## Environment-Specific Builds

### Development vs Production

Use build modes for different environments:

```bash
# Development (larger bundle, includes debug info)
flutter build web --profile

# Production (optimized, minified)
flutter build web --release
```

### Custom Build Flavors

For multiple environments:

```bash
# Build with custom defines
flutter build web \
  --release \
  --dart-define=API_URL=https://api.production.com \
  --dart-define=ANALYTICS_ID=UA-XXXXXX
```

Access in code:
```dart
const apiUrl = String.fromEnvironment('API_URL');
const analyticsId = String.fromEnvironment('ANALYTICS_ID');
```

## Rollback Strategies

### GitHub Pages

```bash
# Revert to previous commit
git revert HEAD
git push origin gh-pages
```

### Firebase Hosting

```bash
# View deployment history
firebase hosting:releases:list

# Rollback to previous version
firebase hosting:rollback
```

### Netlify

Use the Netlify dashboard to rollback to any previous deployment with one click.

### Vercel

In Vercel dashboard, navigate to **Deployments** and promote any previous deployment to production.

## Continuous Deployment

Automate deployments with Git:

<CodeGroup>
```yaml GitHub Actions + Firebase
name: Deploy to Firebase
on:
  push:
    branches: [ main ]

jobs:
  deploy:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v3
      - uses: subosito/flutter-action@v2
        with:
          flutter-version: '3.19.0'
      - run: flutter pub get
      - run: flutter build web --release
      - uses: FirebaseExtended/action-hosting-deploy@v0
        with:
          repoToken: '${{ secrets.GITHUB_TOKEN }}'
          firebaseServiceAccount: '${{ secrets.FIREBASE_SERVICE_ACCOUNT }}'
          projectId: your-project-id
```

```yaml GitHub Actions + Netlify
name: Deploy to Netlify
on:
  push:
    branches: [ main ]

jobs:
  deploy:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v3
      - uses: subosito/flutter-action@v2
      - run: flutter pub get
      - run: flutter build web --release
      - uses: netlify/actions/cli@master
        with:
          args: deploy --prod --dir=build/web
        env:
          NETLIFY_AUTH_TOKEN: ${{ secrets.NETLIFY_AUTH_TOKEN }}
          NETLIFY_SITE_ID: ${{ secrets.NETLIFY_SITE_ID }}
```
</CodeGroup>

## Troubleshooting

### Routes Not Working

Ensure your hosting platform redirects all routes to `index.html`. Check the rewrite rules in your configuration.

### Assets Not Loading

Verify `--base-href` matches your deployment path:
- Root domain: `--base-href "/"`
- Subdirectory: `--base-href "/subdir/"`

### PWA Not Installing

Check:
1. HTTPS is enabled (required for PWA)
2. `manifest.json` is accessible
3. Service worker is registered
4. Icons are available at specified paths

### Large Initial Load Time

Optimize with:
```bash
flutter build web --web-renderer html --tree-shake-icons
```

Or use code splitting (Flutter 3.10+).
